# Code of Conduct

## Our Standard

We want this project to feel calm, practical, and respectful.

Examples of behavior that help:

- assuming good intent
- giving actionable technical feedback
- being kind to new contributors
- disagreeing without hostility
- documenting tradeoffs clearly

Examples of behavior that are not acceptable:

- harassment or intimidation
- insulting or demeaning language
- bad-faith personal attacks
- publishing private information without permission
- repeatedly derailing technical discussions

## Enforcement

If you need to report a conduct issue, please contact the repository maintainer through a private GitHub channel rather than opening a public issue.

## Scope

This code of conduct applies to project discussions, issues, pull requests, and other project spaces.
