# Agent Control Plane v0.1.0

The first public release of Agent Control Plane turns an internal working pattern into a reusable open-source scaffold.

## Highlights

- file-based coordination for multi-agent local workspaces
- generated worker registry from worker body files
- portable `_agent_profile` snapshots for migration and audit
- example workspaces for local workers and project workers
- Python CLI with tests
- a durable memory management model built on worker body files
- initial visual direction for a management dashboard, desktop control chat, and mobile control chat

## Why This Release Matters

Most agent setups invest heavily in prompts and too little in operations.

This release focuses on the operational layer:

- how workers are defined
- how they are discovered
- how governance survives context loss
- how setups move across machines
- how memory becomes durable infrastructure instead of temporary chat state

## Included in v0.1.0

- `agent-control-plane sync-registry`
- `agent-control-plane snapshot-profiles`
- worker profile schema and template
- example worker bodies
- coordination prompt scaffold
- basic CI-ready test coverage
- memory-oriented worker body structure
- README-level dashboard, desktop chat, and mobile UI screenshots

## Early Limitations

- no dedicated validation subcommand yet
- no UI yet
- no task/result helper generator yet
- governance conventions are intentionally lightweight

## Next Likely Steps

- validation commands
- richer task and handoff helpers
- a real management dashboard
- a real desktop operator chat
- a mobile control page and chat UI
- stronger schema enforcement
